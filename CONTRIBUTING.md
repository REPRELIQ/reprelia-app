# Guia de Contribuição

Obrigado por considerar contribuir com este projeto.

## Fluxo de trabalho

1. Crie um fork ou branch a partir de `main` usando o padrão `tipo/descricao-curta` (ex.: `feat/login-oauth`, `fix/erro-validacao`).
2. Faça commits seguindo [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `test:`).
3. Garanta que os testes e linters passam antes de abrir o Pull Request.
4. Abra um Pull Request para `main` usando o template fornecido, descrevendo claramente a mudança e seu motivo.
5. Aguarde revisão de código antes do merge — pelo menos uma aprovação é necessária.

## Padrões de qualidade

- Código deve ter cobertura de testes para novas funcionalidades e correções de bugs.
- Evite introduzir dependências desnecessárias.
- Siga as convenções de estilo já presentes no projeto.

## Relatando bugs ou sugerindo melhorias

Use os templates de issue disponíveis em `.github/ISSUE_TEMPLATE/`.

## Código de Conduta

Ao contribuir, você concorda em seguir o [Código de Conduta](CODE_OF_CONDUCT.md) deste projeto.
